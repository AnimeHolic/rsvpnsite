Privacy Policy

PDFUnlocker Privacy Policy

Last Updated: July 28, 2025

1. Introduction

PDFUnlocker is a mobile application designed to remove passwords from PDF files on Android devices. Our mission is to provide a simple and secure tool to unlock password-protected PDFs while prioritizing your privacy. This Privacy Policy outlines how we collect, use, and protect your data, as well as the purpose of the permissions required by the app.

2. Application Functionality

PDFUnlocker offers the following features to help you manage password-protected PDF files:

  • File Selection: Select a PDF file from your device's storage to check if it is password-protected.
  • Password Removal: If the PDF is password-protected, you can enter the password to unlock it. The app removes both user and owner passwords, creating an unprotected copy of the file.
  • File Saving: Unlocked PDFs are saved to a designated folder (/storage/emulated/0/UnlockedPDFs) with the prefix unlocked_ added to the file name. Non-protected PDFs can be copied to this folder as is.
  • User Feedback: The app provides clear feedback through dialogs and snackbars, informing you of success (e.g., "PDF unlocked successfully!") or errors (e.g., incorrect password or permission issues).
  • Privacy and About Information: Access a privacy policy and usage guide within the app to understand how it works and how your data is handled.

All processing occurs locally on your device, ensuring that your PDF files and their contents remain private and are not uploaded to any external servers.

3. Data We Collect

PDFUnlocker is designed to minimize data collection and protect your privacy. The app only accesses the following data:

  • PDF Files: The app accesses PDF files you explicitly select to check for password protection, remove passwords, and save the unlocked versions.
  • No Personal Information: We do not collect personally identifiable information such as your name, email, location, contacts, or device identifiers.

4. Permissions and Their Purpose

PDFUnlocker requires specific permissions to function properly, as defined in the Android manifest. Below is a detailed explanation of each permission and why it is necessary:

  • READ_EXTERNAL_STORAGE (Android SDK ≤ 33): This permission allows the app to read the PDF file you select from your device's storage. It is required to access the file's contents and check if it is password-protected. Without this permission, the app cannot open the selected PDF file for processing.
  • WRITE_EXTERNAL_STORAGE (Android SDK ≤ 33): This permission enables the app to save the unlocked PDF file to the /storage/emulated/0/UnlockedPDFs folder on your device. It is necessary to create and store the new file after password removal or copying a non-protected PDF.
  • MANAGE_EXTERNAL_STORAGE (Android SDK > 29): For Android devices running SDK versions above 29 (Android 10 and later), this permission is required to read and write files in external storage due to Android's Scoped Storage framework. It allows the app to access and save files in the /storage/emulated/0/UnlockedPDFs folder, ensuring compatibility with modern Android versions. This permission is critical for the app to function on newer devices.

These permissions are used solely for the app's core functionality of selecting, processing, and saving PDF files. The app checks the Android SDK version to request the appropriate permission, ensuring compatibility across different Android versions. If permissions are denied, the app displays a dialog explaining why they are needed and guides you to grant them or open app settings if permanently denied.

5. How We Use Your Data

The data accessed by PDFUnlocker (i.e., PDF files) is processed locally on your device to perform the following tasks:

  • Check Password Protection: The app reads the selected PDF to determine if it is password-protected.
  • Password Removal: If a password is provided, the app uses it to unlock the PDF and remove its security settings.
  • File Saving: The unlocked or copied PDF is saved to the /storage/emulated/0/UnlockedPDFs folder.

No data is uploaded, transmitted, or stored on external servers. All operations are performed locally to ensure your privacy.

6. Data Security

We prioritize the security of your data. All PDF processing is performed locally on your device, and the app does not require an internet connection for its core functionality. This ensures that your PDF files and their contents remain private and are not shared with any external entities.

7. Third-Party Libraries

PDFUnlocker uses the following third-party libraries to provide its functionality:

  • Syncfusion Flutter PDF: Processes PDF files to check for password protection and remove passwords.
  • File Picker: Enables you to select PDF files from your device's storage.
  • Permission Handler: Manages storage permissions to ensure the app can access and save files.
  • Device Info Plus: Retrieves your device's Android SDK version to determine which storage permissions to request, ensuring compatibility across Android versions.

These libraries may collect anonymous usage data for crash reporting or performance improvements, but they do not collect personally identifiable information. Please review the respective privacy policies of these libraries for more details.

8. Your Rights

Since PDFUnlocker processes data locally and does not collect personal information, you have full control over your PDF files. You can delete the files from your device at any time. If you have concerns about your data or the app's functionality, please contact us.
